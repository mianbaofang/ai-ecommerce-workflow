# 调用样例(5 种典型场景)

## 样例 1:最简调用(只给一行)

```
【跑电商新品上架流程】
产品:便携咖啡杯(可折叠,304 钢内胆,容量 350ml)
类目:餐饮具 / 杯具
平台:淘宝
```

**预期输出:** 15 项 + 4 角色,但竞品价格 / 评论会标"待核验"(因为没给已知竞品和价格),第15项会按预算未知输出分档运营计划

---

## 样例 2:带价格和竞品(推荐精度)

```
【跑电商新品上架流程】
产品:便携咖啡杯(可折叠,304 钢内胆,容量 350ml)
类目:餐饮具 / 杯具
平台:淘宝
售价区间:79-129 元
成本:25 元
已知竞品:Stanley(美)/ 酷革(国产) / 哈尔斯(国产)
产品图:已附 4 张
```

**预期输出:** 精度最高,竞品分价格带、差异化机会、主图策划都能落地

---

## 样例 3:多平台(标题/关键词必须分版本)

```
【跑电商新品上架流程】
产品:便携咖啡杯(可折叠)
类目:餐饮具
平台:淘宝 + 亚马逊
售价区间:79-129 元
```

**预期输出:** 标题会出 2 版(淘宝 + 亚马逊),关键词带语言版本提示

---

## 样例 4:类目已有大量竞品(必须分价格带)

```
【跑电商新品上架流程】
产品:智能体脂秤
类目:健康 / 体重秤
平台:淘宝 + 拼多多
售价区间:59-199 元
已知竞品:小米 / 华为 / 云麦 / 斐讯
```

**预期输出:** 竞品分析按 4 个价格带拆(59-89 / 89-129 / 129-199 / 199+),差异化机会偏"专业 + 颜值"组合

---

## 样例 5:无图 / 无详细参数(精度最低)

```
【跑电商新品上架流程】
产品:儿童积木玩具
类目:玩具
平台:淘宝
```

**预期输出:** 模型基于常识跑,所有"竞品价格 / 评论"标"待核验",建议主人先跑一遍生意参谋

---

## ⚠️ 调用禁忌

1. **不要只说"帮我分析一下这个产品"** — 缺输入规范里的 3 个必填项,大管家会反过来问你
2. **不要同时跑 2 个产品** — 一条消息只跑 1 个,跑完再说下一个
3. **不要在调用里夹其他任务** — 比如"跑上架流程 + 帮我查一下 XX 股票" — 会被拒绝
4. **已知竞品给链接优于给名字** — 链接能二次核验价格/评论
5. **产品图尽量一次给全** — 正面 / 背面 / 内部 / 上身 / 细节,少图会降主图策划精度
