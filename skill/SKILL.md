# AI 电商新品上架工作流

> 版本:1.0  
> 定位:电商运营 / 操盘手。把一个新品的"上架前准备"压缩到一次结构化会话内跑完。

## 一、Skill 目标

把"一个新品从零到可上架"的所有前置工作,**串成一条决策链路**,而不是一堆零散素材。

| 阶段 | 输出 | 价值 |
|---|---|---|
| 0. 资料接收 | 结构化产品档案 | 把"丢给 Codex"的那句话变成可执行输入 |
| 1. 战略层 | 产品定位 / 差异化机会 | 先定方向,不写文案 |
| 2. 市场层 | 竞品分价格带 / 机会点 | 找空白带,不拼低价 |
| 3. 用户层 | 评论反推 / 用户语言 | 翻译"厂家语言"→"用户语言" |
| 4. 视觉层 | 主图 5 张 / 详情页结构 | 每张图带"点击逻辑" |
| 5. 流量层 | 标题 / 关键词三层 | 平台分版本,搜索意图分析 |
| 6. 客服层 | FAQ / 客服话术 | 提前拆雷 |
| 7. 协作层 | 设计 Brief / 角色分工 | 运营/设计/客服/老板四角色执行建议 |

## 〇、能力预检与自动安装（执行前必做）

执行任何模块前，必须先检查当前 Agent 环境可用的 companion skills。如果宿主环境支持 Skill 安装（如 OpenClaw / Hermes / 其他 Skill-compatible Agent），缺失的 companion skills 应自动安装；如果不支持，输出缺失清单并降级。

**自动安装步骤：**
1. 检查 `multi-search-engine`、`anysearch`、`firecrawl-search`、`firecrawl-scrape`、`agent-reach` 是否已安装
2. 检查 `humanizer-zh` 是否已安装
3. **如果宿主 Agent 支持 `npx skills add` / `skills install` 等安装命令，对缺失的 skills 执行自动安装**
4. 安装后再验证一次，输出可用/不可用清单
5. 如果宿主环境不支持自动安装，输出推荐安装清单，告知用户粘贴命令安装

**预检命令（伪代码）：**
```text
1. for each skill in [multi-search-engine, anysearch, firecrawl-search, firecrawl-scrape, agent-reach, humanizer-zh]:
2.   if not installed:
3.     if runtime supports auto-install:
4.       run: npx skills add <skill-name>  (或等效安装命令)
5.     else:
6.       mark as missing, add to install-recommendation list
7. 输出预检清单，标注每个模块的依赖状态
```

**预检输出格式：**
```markdown
## 能力预检
| 模块 | 依赖 | 状态 | 缺失影响 |
|---|---|---|---|
| 竞品数据获取 | anysearch/firecrawl/agent-reach/multi-search-engine | ✅ 已安装/⚠️ 缺失 | 竞品价格/评论标注【待核验】 |
| 文案人味化 | humanizer-zh | ✅ 已安装/⚠️ 缺失 | 使用内置规则，精度降低 |
| 生图生成 | 用户提供模型/工具 | 🔧 由用户指定 | 只输出 prompt 和建议 |
| 视频生成 | 用户提供视频工具 | 🔧 由用户指定 | 只输出分镜和 prompt |
```

**预检后执行以下规则：**
- ✅ 已安装 → 执行时自动调用对应 companion skill
- ⚠️ 缺失 → 对应模块标注【待核验】或使用内置兜底规则，不影响主流程完整性
- 🔧 需用户提供 → 只输出 prompt、brief、preflight 问题，不执行实际生成

## 二、输入规范

调用时,主人在一条消息里尽量提供以下资料(缺失的由模型基于常识补全,会标注"待核验"):

### 2.0 运行模式(先判断,再执行)

识别到触发句后,先判断用户想跑哪种模式。用户未指定时,默认使用**完整上架模式**,但必须在验证阶段提示可切换。

| 模式 | 适用场景 | 输出范围 | 预计耗时 |
|---|---|---|---|
| **快速诊断模式** | 用户只想先判断这个品能不能做 | 1/2/4/5/6 + 决策摘要 | 1-3分钟 |
| **完整上架模式** | 从零到可上架的完整准备 | 1-15 + 4角色分工 | 3-8分钟 |
| **素材制作模式** | 运营方向已定,只要主图/详情页/标题/生图prompt | 3/7/8/9/14 + 可选生图/视频 | 2-5分钟 |
| **上架后优化模式** | 已经上架,要复盘和优化 | 5/9/10/11/12/13/15 + 优化清单 | 2-5分钟 |

**模式选择规则:**
- 用户说"先看看值不值得做" → 快速诊断模式
- 用户说"全流程/完整跑/从零上架" → 完整上架模式
- 用户说"主图/详情页/标题/生图/视频" → 素材制作模式
- 用户说"已经上架/数据不好/点击低/转化低" → 上架后优化模式

### 2.0.1 分层交付(强制)

无论哪种模式,输出必须分三层,避免长报告难消费:

1. **一页决策摘要** - 先给结论、机会、风险、下一步。
2. **详细模块输出** - 按所选模式输出对应项。
3. **执行交付包** - 按运营/设计/客服/老板/素材生成拆待办。

**一页决策摘要格式:**

```markdown
## 一页决策摘要
- **建议动作:** [继续打样/补资料/谨慎上架/暂缓]
- **主打方向:** [一句话定位]
- **机会点:** [1-3条]
- **最大风险:** [1-3条]
- **优先做:** [接下来3个动作]
- **不能做:** [需要删掉或避免的方向]
- **待核验:** [必须人工确认的数据]
```



**必填(无则不跑):**
- 产品名称 + 一句话描述
- 产品类目(通勤包 / 家居 / 美妆 / 3C ...)
- 目标平台(淘宝 / 拼多多 / 亚马逊 / 多平台)

**推荐提供(决定精度):**
- 产品图(正面 / 背面 / 内部 / 上身 / 细节)
- 基础参数(尺寸 / 重量 / 材质 / 适配对象)
- 成本区间 + 心理售价
- 已知竞品(品牌名 / 链接 / 价格)
- 已知痛点或卖点(可选)

**调用触发句(任一即可):**
- "帮我跑一下这个新品的 Codex 流程"
- "用电商上架流程分析这个品"
- "新上品,跑上架前准备"

**输入验证流程(识别到触发句后必须执行):**

当识别到触发句后,按以下顺序验证:

1. **检查必填项**
   - ✓ 产品名称 + 一句话描述
   - ✓ 产品类目
   - ✓ 目标平台
   
   **缺失任一必填项时,输出:**
   ```
   ⚠️ 缺少必填信息,无法开始执行
   
   请补充:
   [ ] 产品名称 + 一句话描述
   [ ] 产品类目(如:箱包/通勤包)
   [ ] 目标平台(淘宝/拼多多/亚马逊/多平台)
   
   💡 快速开始:
   1. 复制 templates/user-input-form.md 填写
   2. 或直接口述以上3项,我来格式化
   ```
   停止执行,等待用户补充。

2. **检查推荐项并标注风险**
   必填项齐全后,检查推荐项缺失情况:
   ```
   ✅ 已接收必填信息,准备执行15项标准输出(预计3-8分钟)
   
   ⚠️ 以下信息缺失,会影响输出精度:
   [ ] 产品图 → 主图策划/设计Brief精度降低30%
   [ ] 成本/售价 → 价格带判断标注【待核验】
   [ ] 已知竞品 → 竞品分析基于常识推测,需后期核实
   
   是否继续? (输入 Y 继续 / N 等我补充)
   ```

3. **数据真实性声明(必须前置)**
   用户确认继续后,输出:
   ```
   ⚠️ 数据真实性声明
   
   本次分析中,以下数据将标注【待核验】:
   - 竞品价格/评论(需用生意参谋/蝉妈妈/卖家精灵核实)
   - 关键词搜索量(需用直通车/生意参谋验证)
   - 类目趋势判断(需用行业工具二次确认)
   
   AI生成的图片仅作创意参考,上架图必须:
   ✓ 实拍产品图
   ✓ 有授权的素材
   ✓ 符合平台规范(无夸大PS/无违禁词)
   
   开始执行 ↓
   ```

4. **开始执行15项输出**
   每完成1项,输出进度:
   ```
   ✅ 1/15 产品定位 已完成
   ⏳ 2/15 目标用户画像 执行中...
   ```

### 2.1 证据等级与可信度(准确性规则)

每个关键结论必须标注证据等级和可信度,不能只写【待核验】。

| 等级 | 来源 | 可用性 | 标注方式 |
|---|---|---|---|
| **A** | 用户明确提供的资料/图片/参数/截图/后台导出 | 可进入草稿,上架前核对 | 【A-用户提供】 |
| **B** | 授权工具导出、公开页面可观察信息、用户提供链接经工具核验 | 可作为强参考,仍需复核 | 【B-已核验】 |
| **C** | 行业常识/类目经验推断 | 只作方向参考 | 【C-经验推断】 |
| **D** | 信息不足下的强假设 | 不可直接执行 | 【D-强假设】 |

| 可信度 | 含义 | 使用方式 |
|---|---|---|
| **高** | 有明确资料或多来源支持 | 可进入执行brief |
| **中** | 逻辑合理但缺数据 | 可作草稿,执行前复核 |
| **低** | 信息不足或依赖假设 | 只作讨论方向 |

**必须标注证据等级的模块:**
- 第1项价格带定位
- 第5项竞品价格带
- 第6项差异化机会
- 第9项标题关键词判断
- 第11项评论洞察
- 第15项运营计划中的目标数据

**格式示例:**
```markdown
- 价格带建议:109-159元 【C-经验推断 / 可信度:中】
  为什么:用户给了成本45元,但未给实际成交价和竞品链接,需要用生意参谋核成交价。
```

**来源迹规则(强制):**
每个涉及竞品、价格、销量、评论、关键词、功效、认证、材质来源的结论,必须附带来源迹:
```markdown
[来源迹: 观察路径 + 时间 + 口径]
```
| 来源类型 | 来源迹格式示例 |
|---|---|
| 公开链接观察 | `[来源: https://item.taobao.com/xxx 商品页公开展示价, 2026-07-04 15:30, 标价109-159元, 未确认券后价]` |
| 搜索工具发现 | `[来源: anysearch 搜索"通勤电脑包 15.6寸", 2026-07-04, 得到候选竞品 A/B/C, 价格未核验]` |
| 授权工具导出 | `[来源: 生意参谋截图(用户提供), 2026-07-04, 行业成交均价 89-129元]` |
| 经验推断 | `[来源: C-经验推断, 类目经验+用户成本45元, 推断价格带 109-159元]` |
| 关联强假设 | `[来源: D-强假设, 无任何数据支撑]` |

没有来源迹的结论不得进入 B 级或以上证据。仅凭模型常识生成的结论统一标注 C 或 D。

**待核验清单(强制输出):**
```markdown
## 待核验清单
| 信息 | 当前结论 | 证据等级 | 需要谁核 | 核验工具/方式 | 不核的风险 |
|---|---|---|---|---|---|
| 竞品成交价 | [结论] | [A/B/C/D] | 运营 | 授权后台/生意参谋/蝉妈妈/卖家精灵/Keepa/用户截图 | 价格带判断错 |
```

### 2.1.1 竞品价格来源规则(强制)

竞品价格不能默认写成"已自动获取真实价格"。不同工具拿到的是不同层级的数据,必须区分:

| 来源 | 可得到什么 | 证据等级 | 使用边界 |
|---|---|---|---|
| 用户截图/表格/后台导出 | 标价、券后价、成交价、销量、评价等用户已掌握数据 | A 或 B | 最适合进入价格带判断,但仍需记录时间和平台 |
| 授权数据工具 | 生意参谋、蝉妈妈、飞瓜、卖家精灵、Keepa、Jungle Scout、Helium 10 等导出数据 | B | 可作为强参考,需说明工具名、时间范围和口径 |
| 公开页面读取 | 商品页展示价、活动价、SKU 可见价、评论摘要 | B 或 C | 只能叫"页面观察价",不能等同真实成交价或最终到手价 |
| 搜索/聚合结果 | 候选竞品、标题、摘要、可能的价格片段 | C | 用于发现竞品和构建候选池,不能直接当价格证据 |
| 模型常识推断 | 常见价格带、类目经验 | C 或 D | 必须标注【待核验】,不可直接执行 |

**Companion data skills(默认自动调用):**
- 本 Skill 设计为可自动调用数据获取 companion skills。执行第5项竞品分析、第9项标题关键词、第11项评论洞察前,应先检查当前 Agent 是否已安装并可用这些 skills。
- 可用时自动调用,不可用时不要中断主流程,但必须在输出中写明"本次缺失的数据获取 skill"和降级影响。
- 开源包不内置任何 API key;需要 key/cookie/登录态的工具由用户或宿主 Agent 按对应 skill 说明配置。

| Companion skill / 工具 | 默认用途 | 输出进入哪类证据 | 不能替代什么 |
|---|---|---|---|
| `multi-search-engine` | 多搜索引擎并行发现候选竞品、品牌页、测评页、平台公开页 | C,可辅助 B | 真实成交价、登录态后台数据 |
| `anysearch` | 全网搜索、候选竞品发现、公开页面摘要、跨来源搜索 | C,可辅助 B | 真实成交价、登录后价格、平台后台数据 |
| `firecrawl-search` | 搜索公开网页并可附带抓取结果 | C,可辅助 B | 平台授权数据、App 内价格、券后价 |
| `firecrawl-scrape` | 读取指定公开 URL 的 markdown/html/links/screenshot | B 或 C | 登录态页面、强反爬页面、完整 SKU 弹层价格 |
| `agent-reach` | Exa 搜索、网页阅读、社媒/内容平台观察、小红书/B站/Twitter/Reddit 等讨论 | C,可辅助 B | 电商后台成交价、平台商业数据 |
| `Tavily` | 通用搜索、网页摘要、问答式资料检索(若用户环境已安装) | C,可辅助 B | 授权后台数据和真实成交价 |
| 浏览器/Playwright 类工具 | 打开公开页面、截图、观察动态页面 | B 或 C | 批量稳定抓价、平台授权数据 |
| 用户提供的后台/第三方工具导出 | 成交价、券后价、销量、评价、关键词、趋势 | A 或 B | 需记录来源、时间和口径 |

**自动调用顺序:**
1. 用 `multi-search-engine` / `anysearch` / `firecrawl-search` / `Tavily` 搜索类目词、品牌词、核心属性词,得到候选竞品池。
2. 对候选 URL 用 `firecrawl-scrape` / `web_fetch` / 浏览器读取公开页面,提取标题、公开展示价、SKU可见信息、卖点、评论摘要。
3. 用 `agent-reach` 补社媒和内容平台观察,尤其是小红书/B站/Reddit/Twitter 等用户反馈、种草话术和痛点表达。
4. 如果用户提供生意参谋、蝉妈妈、飞瓜、卖家精灵、Keepa、Jungle Scout、Helium 10 等截图或导出,优先使用这些数据校正价格带。
5. 汇总时输出"数据来源表":每个竞品至少写来源、工具、链接/截图、价格口径、读取时间、证据等级。

**缺失处理:**
- 如果 companion skills 未安装,输出推荐安装清单:`multi-search-engine`、`anysearch`、`firecrawl-search`、`firecrawl-scrape`、`agent-reach`;Tavily 作为可选搜索 API。
- 缺 `firecrawl-scrape` / `web_fetch` 时,搜索结果只能作为候选发现,不能当页面核验。
- 缺 `agent-reach` 时,社媒/内容平台洞察降级为普通搜索结果。
- 缺授权工具或用户导出时,成交价/券后价/销量统一标注【待核验】。

- 上述搜索/抓取类工具负责"自动发现候选"和"读取公开观察值";授权工具、用户截图或导出表才是价格/销量关键数据的优先来源。

**价格表述规则:**
- 能读到公开页面时,写"页面观察价"或"公开展示价",并记录来源链接和读取时间。
- 看不到 SKU、券后价、运费、地区价时,不得写"真实成交价"。
- 多平台比价时,必须说明口径差异:标价、券后价、直播间价、会员价、含/不含运费。
- 无法核验时,第5项只能输出"价格带假设",并进入待核验清单。

## 三、15 项标准输出(强约束,顺序执行)

执行时**必须**按此顺序输出,缺一项视为未完成:

1. **产品定位** — 解决什么问题 / 放在哪个价格带
2. **目标用户画像** — 谁买、买给谁、决策路径
3. **核心卖点** — 3-5 个,每个带"用户语言翻译"
4. **用户痛点** — 真实使用场景里的烦
5. **竞品分析(分价格带)** — 至少 3 个价格带,标注打法与问题
6. **差异化机会** — 至少 3 个可打的细分方向(必须符合判断框架)
7. **主图策划(5 张)** — 每张带:画面 / 文案 / 点击逻辑 + **5套差异化方案选择**
8. **详情页结构** — 按"用户下单前疑虑顺序"排列 + **5屏标准结构**

## 四.四、第8项执行标准:详情页5屏标准结构

**详情页不是卖点堆砌,而是顺应用户决策心理的"说服路径"。必须按5屏结构输出:**

### 5屏标准结构(强制顺序)

| 屏数 | 功能名称 | 内容要求 | 心理目标 | 停留时长 |
|---|---|---|---|---|
| **第一屏** | 心智定位 | 一句话定义产品+悬浮大图 | 快速吸引注意力,建立第一印象 | 3秒内决定是否继续看 |

| **第四屏** | 场景延伸 | 拼图展示多个使用场景(上班/上课/出游) | 增加使用频次预期,扩大适用人群 | 8-10秒 |

### 15. 30天运营推进计划

#### 7天测款期(新品冷启动)

| 天数 | 关键动作 | 检查点 | 目标数据 | 失败重试方案 |
|---|---|---|---|---|
| D1 | 基础素材检查 | 主图/详情页/标题是否符合规范 | 无违规 | 发现违规词立即替换 |
| D2-D3 | 主图点击率测试 | 5套主图方案小流量测试(每套100-200曝光) | 确定点击率最高的1套 | 若全部<2%,返回重做主图 |
| D4-D5 | 引导首批真实评价 | 引导5-10条真实买家秀(带图) | 建立初始信任 | 若无人下单,检查价格/详情页 |
| D6-D7 | 投放精准词+分析跳失 | 详情页哪一屏跳失率最高 | 找到优化方向 | 若跳失率>70%,优化第X屏 |

**测款期判断标准:**
- ✅ 通过:主图点击率>2%,详情页跳失率<60%,有5条以上评价 → 进入起量期
- ⚠️ 观察:主图点击率1-2%,跳失率60-70% → 优化后继续测试3天
- ❌ 放弃:主图点击率<1%且优化2次仍无改善 → 考虑换品或重新定位

#### 14天起量期(流量起步)

| 天数 | 关键动作 | 检查点 | 目标数据 | 执行细节 |
|---|---|---|---|---|
| D8-D10 | 标题AB测试 | 从20个标题池中测试3-5个 | 确定高转化标题 | 每个标题测试2天,记录搜索曝光和点击 |
| D11-D12 | 积累高质量评价 | 10-20条评价,至少5条买家秀 | 转化率提升5-10% | 主动联系好评用户索要买家秀 |
| D13 | 关键词顺序优化 | 根据搜索反馈调整关键词顺序 | 搜索排名提升 | 把转化高的关键词前置 |
| D14 | 客服话术同步 | 高转化卖点同步给客服话术库 | 咨询转化率提升 | 整理FAQ前5的问题,标准化回复 |

**起量期判断标准:**
- ✅ 通过:日销量>5单,转化率>3%,评价>10条 → 进入稳定期
- ⚠️ 观察:日销量3-5单,转化率2-3% → 加大直通车投入
- ❌ 调整:日销量<3单 → 检查价格竞争力/详情页/主图

#### 30天稳定期(持续优化)

| 天数 | 关键动作 | 检查点 | 目标数据 | 拓展方向 |
|---|---|---|---|---|
| D15-D20 | 固定高转化素材 | 主图/标题不再变动,详情页微调 | 稳定转化率 | 数据稳定后才能做AB测试 |
| D21-D23 | 详情页二次优化 | 根据跳失率优化第X屏(热力图分析) | 跳失率降低5-10% | 重点优化跳失率>20%的屏 |
| D24-D26 | 制作种草素材 | 短视频/小红书图文笔记 | 站外引流 | 找达人合作或自己拍摄 |
| D27-D28 | 优惠券AB测试 | 测试不同券面组合(满减/折扣) | 客单价优化 | 记录哪种优惠券转化率最高 |
| D29-D30 | 建立竞品监控机制 | 每周记录竞品价格/主图/评论变化 | 持续优化依据 | 用表格记录,发现竞品动作及时跟进 |

**稳定期判断标准:**
- ✅ 成功:日销量>10单,ROI>2,复购率>5% → 考虑加大投入或拓展SKU
- ⚠️ 维持:日销量5-10单,ROI 1.5-2 → 继续优化,控制成本
- ❌ 衰退:日销量<5单且持续下滑 → 分析原因(竞品降价/评价下降/平台流量变化)

### 预算适配与止损规则(强制)

**默认策略:未提供预算时,只输出免费/低预算方案(<100元/天),不做付费投流计划。**

#### 预算档位
| 日预算 | 7天测款期重点 | 14天起量期重点 | 30天稳定期重点 |
|---|---|---|---|
| **≤100元(或无预算)** | 主图测试+精准词(免费/自然流量) | 标题优化+评价积累 | 详情页优化 |
| **100-500元** | 精准词测试+人群测试+主图AB | 场景词拓展+评价积累 | 站外种草+AB测试 |
| **500元+** | 精准词+场景词+达人合作 | 直播测试+多词拓展 | 竞品监控+多计划测试 |

#### Kill Switch 规则(每项投流建议必须附带)

任何涉及付费流量的建议,必须同时输出以下停止条件:

```markdown
- 日预算上限: [数字]元
- 单次点击成本上限(CPC): [数字]元,超过暂停
- 投产比下限(ROI): [数字]以下暂停,持续3天未达标则关停计划
- 关键指标: [点击率/转化率/加购率,选1-2个]
- 停止动作: [暂停计划/更换素材/降低出价/关停]
- 检查周期: [每天/每3天/每周]
```

**示例:**
```markdown
- 日预算上限: 100元
- CPC上限: 2元,超过暂停
- ROI下限: 1.5,以下暂停,连续3天未达标关停
- 关键指标: 点击率>3%
- 停止动作: 暂停计划→换主图→降低出价
- 检查周期: 每天检查
```

#### 无预算约束
用户没有提供预算时,默认按"≤100元"执行,并在计划开头输出:
```
⚠️ 未提供预算,默认按低预算测款方案执行。
如需付费投流,请补充日预算金额。
```

#### 投产建议约束
1. 投流建议必须附带 Kill Switch,缺少停止条件的投流计划视为无效。
2. 测款期(D1-D7)默认禁止单日消耗>预算50%,禁止连续消耗超过跑款周期3天。
3. D8-D14首次出现ROI<1.0,自动转入"只优化不消耗"模式。
4. 任何"加大投放""拓展词"必须附带明确的数据前提(如"CTR>5%持续3天才加大")。
```

## 四.七、文案人味化质检(融合 humanizer-zh 原则)

**所有用户可见文案必须经过"人味化"处理。** 适用范围包括:
- 第3项核心卖点的用户语言
- 第7项主图文案
- 第8项详情页文案
- 第9项标题测试池
- 第12项FAQ
- 第13项客服话术
- 第14项设计Brief中的文案层级
- 第15项运营计划中的客服/评价/种草话术

### 可选依赖

如本地安装了公开 skill `humanizer-zh`,可在最终输出前调用它做二次改写。没有安装时,按本节规则内置执行,不得中断主流程。

### 5条核心原则

1. **删除填充短语** - 去掉"此外""值得注意的是""在当今时代"等空转连接词。
2. **打破公式结构** - 少用"不仅...更是...""从X到Y""三段式排比"。
3. **变化节奏** - 主图文案短,详情页文案长短交错,客服话术像真人回复。
4. **信任读者** - 直接说事实和场景,不要解释一堆"为什么重要"。
5. **删除金句感** - 听起来像广告口号但没有具体信息的句子必须重写。

### 电商文案合规拦截门(强制,优先级高于人味化)

**所有用户可见文案输出前,必须先过合规拦截门。** 检测到以下列表中的任何表达→**拒绝输出该段**→标注违规项→强制改写→通过后再进入人味化。

#### 平台通用绝对禁用词(检测到直接拦截,不得输出)
- 商标法范畴: **最**、**第一**、**顶级**、**首个**、**首创**、**唯一**、**首家**、**极品**、**绝佳**、**100%**、**纯天然**、**零添加**、**无副作用**、**永不**、**永久**
- 广告法范畴: **国家级**、**国家**、**国级**、**世界级**、**全网**、**全国**、**全球**、**第一品牌**、**领导者**、**冠军**、**金牌**
- 功效/安全范畴: **根治**、**治愈**、**痊愈**、**特效**、**万能**、**排毒**、**不反弹**、**轻松减肥**、**快速瘦身**、**美白**、**祛斑**、**消炎**、**抗菌**
- 平台规则范畴: **正品保证**(需要授权资质)、**假一赔十**(需要平台开通)、**支持专柜验货**(多数品牌不支持)、**七天无理由**(需符合平台规则)

#### 平台特定规则
| 平台 | 标题限制 | 额外禁词 |
|---|---|---|
| 淘宝 | 30汉字内,前8字核心词 | 禁止使用"淘宝网"、"天猫"等平台名作关键词,禁止"最"、"第一"、"全网" |
| 拼多多 | 标题含价格词更有效 | 禁止虚假拼单、"已售XX万"无数据支撑 |
| 抖音 | 场景化+口语化 | 禁止虚假宣传、"不好用退钱"无边界 |
| 亚马逊 | 200字符内,禁止促销词 | 禁止"Best"、"#1"、"Guaranteed"无资质、"EC"认证无证明 |
| 快手 | 口语化+信任感 | 禁止绝对承诺、功效违规、未成年人形象 |
| 1688 | 参数+规格优先 | 禁止虚标起批量、虚假资质 |

#### 合规检测流程(每个文案模块输出前执行)
```markdown
1. 扫描全文,命中任何绝对禁用词 → 整段退回,标注"违规项: [具体词]",强制改写
2. 扫描全文,命中任何平台特定禁词 → 根据目标平台判断:该平台禁则退回,非禁则保留但标注
3. 涉及功效/认证/安全/检测报告 → 必须标注【人工复核】,不得直接输出
4. 用户没有提供授权资质文件 → 不得写入"通过XX认证/符合XX标准/经XX检测"
5. 通过检测后才能进入人味化质检
```

### 改写标准

**主图文案:**
- 8-14字优先,一句只说一个点击理由。
- 不写"高级大容量",写"电脑衣物分开放"。
- 不写"舒适减负",写"肩带加宽不勒肩"。

**详情页文案:**
- 每屏只回答一个疑虑。
- 先写用户场景,再写产品解决方式。
- 用实拍、尺寸、对比、评价来支撑,不要靠形容词堆砌。

**客服话术:**
- 像真人客服,不要像公告。
- 先接住问题,再给事实,最后给下一步动作。
- 不承诺无法兑现的结果。

**标题:**
- 标题允许关键词密度,但不能重复词硬堆。
- 测试池里每个标题必须有不同的搜索意图,不能只换顺序。

### 输出前人味化自检

每个文案模块输出前检查:
- [ ] 是否删掉了"此外/值得注意的是/综上所述"等连接词?
- [ ] 是否避免"不仅...更是..."和三段式口号?
- [ ] 是否用具体细节替代"高级感/品质感"?
- [ ] 是否把夸大承诺改成可验证、有边界的表达?
- [ ] 客服话术是否像真人回复,而不是品牌公告?

### 示例

**改写前:**
> 这款通勤包不仅拥有大容量设计,更是现代职场人士提升通勤体验的重要选择,展现高级品质感。

**改写后:**
> 电脑、文件和换洗衣物分开放。上班背不乱,短途出差也不用再拉箱子。

## 四.八、生图前交互模块(开源版抽象)

**本 skill 不绑定任何私有生图工具。** 如果用户需要生成主图创意图、详情页场景图或对比图,必须先让用户选择自己的生图模型/工具,再确认参数。不得默认调用任何模型。

### 什么时候询问生图

在第7项主图策划、第8项详情页结构或第14项设计Brief完成后,如用户输入包含"生成图/生图/做主图/出效果图/给设计参考"等意图,或用户明确要求,才进入生图交互。

### 必问信息

```text
是否需要生成创意参考图? 此图只作设计沟通,不能直接上架。

请确认:
1. 生图工具/模型: [用户自选,如 Midjourney / Stable Diffusion / GPT Image / 即梦 / 通义万相 / 其他]
2. 图片用途: 主图1 / 主图2-5 / 详情页场景图 / 对比图 / 细节图 / 海报图
3. 参考图: 是否有产品实拍图? 有几张? 哪张是主参考?
4. 比例尺寸: 1:1 / 4:3 / 3:4 / 16:9 / 9:16 / 平台指定尺寸
5. 张数: 1 / 2 / 4 / 其他
6. 风格: 白底电商 / 写实场景 / 轻商务 / 小红书种草 / 信息图 / 对比图
7. 是否允许出现文字: 是/否;如是,文字必须由设计后期排版更稳
8. 保存路径或交付形式: 本地路径 / prompt文本 / 交给设计
```

**用户没有确认模型/工具和参数前,不得进入生成。**

### 生图边界

- AI图只作创意参考,真实上架图必须用实拍图或授权素材。
- 产品外观、颜色、结构不得被AI擅自改造。
- 涉及商标、认证、材质、容量、功效的内容必须人工复核。
- 不生成虚假检测报告、虚假认证、虚假用户评价截图。
- 不在AI图中直接生成复杂中文卖点;文字建议后期排版。

输出生图 prompt 时,必须按"用途 + 主体 + 场景 + 构图 + 光线 + 风格 + 禁止项 + 后期说明"结构写。中文为主,可附英文版供模型使用。

```text

风格: [白底电商/轻商务/生活方式/信息图]

```

## 四.十、产品渲染视频可选模块

产品渲染视频不是每个新品都必须做。只有当产品需要动态解释、场景种草或投放短视频时,才进入视频分支。

### 什么时候需要视频

| 场景 | 是否建议 | 理由 |
|---|---|---|
| 结构复杂、需要演示 | 建议 | 静态图讲不清收纳/安装/折叠/使用步骤 |
| 抖音/快手/小红书投放 | 建议 | 需要前3秒钩子和动态场景 |
| 高客单价新品 | 可选 | 用视频建立质感和信任 |
| 低价标品且主图足够 | 可不做 | 先用主图和详情页测款 |
| 预算不足或无素材 | 暂缓 | 先完成上架基础素材 |

### 视频前置询问

```text
是否需要产品渲染/种草视频? 如需要,请确认:
1. 视频用途: 主图视频 / 详情页视频 / 抖音短视频 / 小红书种草 / 投放素材
2. 视频类型: 产品展示 / 使用场景 / 功能演示 / 对比转化 / 30天测款素材
3. 时长: 6秒 / 10秒 / 15秒 / 30秒 / 其他
4. 比例: 1:1 / 16:9 / 9:16
5. 素材: 产品图 / 实拍视频 / LOGO / 卖点文案 / 禁用内容
6. 风格: 棚拍质感 / 生活方式 / 信息流广告 / 小红书自然分享
7. 是否需要旁白: 是/否
8. 输出: 只要脚本 / 分镜表 / 视频生成prompt / 完整制作brief
```

### 视频输出结构

如用户选择生成视频方案,输出:

1. **视频定位** - 解决什么转化问题。
2. **前3秒钩子** - 让用户停下来的一句话或画面。
3. **分镜脚本** - 时间轴、画面、动作、字幕、卖点。
4. **素材清单** - 需要哪些产品图/实拍/场景/音效。
5. **视频 Prompt** - 给用户选择的视频工具使用。
6. **风险提示** - 哪些画面必须实拍,哪些不能AI虚构。

### 视频 Prompt 结构

```text
视频用途: [主图视频/信息流广告/种草短视频]
时长比例: [10秒,9:16]
主体: [产品名称,外观必须忠于参考图]
镜头1 0-3秒: [停留钩子画面]
镜头2 3-6秒: [核心卖点演示]
镜头3 6-10秒: [场景/对比/收束]
字幕: [短句,每屏不超过12字]
运镜: [轻推近/固定/横移/俯拍,不夸张]
风格: [真实电商短视频,自然光,干净背景]
禁止项: [不改变产品结构,不虚构功能,不出现未授权logo]
```

## 五、与其他 skill/工具的协同

**工具调用优先级(自动判断):**

### 5.1 竞品数据获取(第5项执行时)

执行第5项时默认自动调用已安装 companion skills,不要只凭模型常识写竞品。

1. **候选发现(自动):** 用 `multi-search-engine` / `anysearch` / `firecrawl-search` / `Tavily` 搜索类目词、品牌词、核心属性词。
   ```
   示例:搜索"通勤双肩包 15.6寸 电脑包 价格 测评",得到候选品牌、商品页、测评页、平台页。
   ```
2. **页面读取(自动):** 对候选 URL 用 `firecrawl-scrape` / `web_fetch` / 浏览器读取公开页面观察值。
   ```
   示例:提取标题、公开展示价、SKU可见信息、主图卖点、评论摘要;标注为"页面观察价"。
   ```
3. **内容平台洞察(自动):** 用 `agent-reach` 补小红书/B站/Reddit/Twitter 等讨论、种草话术和痛点表达。
   ```
   示例:搜索"通勤电脑包 压肩 差评"、"commuter backpack laptop review"。
   ```
4. **用户/授权数据校正(优先):** 如用户提供截图、表格、后台导出或授权工具数据,优先用它校正价格带。
   ```
   示例:用户提供"竞品价格表/生意参谋截图/蝉妈妈导出/Keepa曲线"。
   ```
5. **缺工具兜底:** 如果相关 companion skills 不可用,输出缺失清单和降级说明,再基于常识生成价格带假设。
   ```
   【待核验-缺少 companion search/scrape skills 或授权数据】
   ```

### 5.2 文案人味化(第3/7/8/9/12/13/14项完成后)
- **优先:** 如用户本地安装公开 skill `humanizer-zh`,可调用它对主图文案、详情页文案、FAQ、客服话术做二次去AI味。
- **兜底:** 未安装 `humanizer-zh` 时,按"四.七 文案人味化质检"内置规则执行。
- **约束:** 人味化不能改变事实参数、价格、材质、售后承诺;只能调整表达方式。

### 5.3 主图/详情页创意图生成(第7/8/14项完成后,可选) 【🔧 需用户提供】
- 开源版**不包含**、也不默认调用任何生图工具。用户必须显式选择自己的模型/工具后才能执行生成。
- 进入生成前必须确认:模型/工具、用途、参考图、比例尺寸、张数、风格、文字策略、保存路径或交付形式。
- 用户未确认前,只输出 prompt 和参数建议,不得调用生成。
- 生成时必须注明: **AI图仅作创意参考,不可直接上架。**
- 使用产品图作 reference 时,必须要求模型忠于产品外观,不得改结构、颜色、logo、材质。

### 5.4 产品渲染/种草视频(第15项或用户要求时,可选) 【🔧 需用户提供】
- 开源版**不包含**、也不默认调用任何视频生成工具。用户必须显式选择自己的视频工具/制作方式后才能执行生成。
- 进入视频方案前必须确认:用途、类型、时长、比例、素材、风格、旁白、输出形式。
- 低预算测款阶段默认只输出视频脚本/分镜/prompt,不默认执行生成。
- 结构复杂品、短视频投放、高客单价新品优先建议视频;低价标品且主图足够时可暂缓。

### 5.5 数据落库(全部完成后,可选)
- 如已加载 `feishu_bitable` / `feishu_create_doc` → 询问用户:
  ```
  ✅ 15项+4角色已全部完成,是否导出到飞书?
  [1] 多维表格(适合数据对比)
  [2] 飞书文档(适合团队协作)
  [N] 不导出
  ```

**工具缺失时的处理:**
- 如无 `web_fetch`/`web_search` 能力 → 竞品分析全部标注【待核验】
- 如无 `humanizer-zh` → 使用内置文案人味化规则

- 工具缺失不影响15项标准输出完整性

## 六、风险与边界

- **数据真实为底线。** 模型基于常识生成的竞品价格 / 评论**必须标注"待核验"**,主人要在上架前自己跑一遍生意参谋 / 蝉妈妈 / 卖家精灵。
- **不做操盘建议。** 不出"先亏 100 单冲量""直接卡首焦"这种话术。
- **不替代运营拍板。** 不知道库存 / 供应链 / 老板预算,这些需要运营最终判断。
- **生成图不直接上架。** AI 出图仅作创意参考,真实上架图必须实拍或授权素材。
- **不绑定私有生图/视频工具。** 开源版只定义交互、prompt和质量规范;具体模型由用户选择。
- **文案必须去AI味。** 不输出空泛口号、模板腔、虚假权威和夸大承诺。

## 十二、执行时常见问题与处理

### 问题1:图片识别偏差
**现象:** AI无法准确识别材质、结构、颜色或细节。  
**原因:** 图片背景杂乱、光线不足、角度遮挡,或产品关键参数未被文字补充。  
**处理:**
- 优先提供白底图、高清细节图、上身图、内部结构图。
- 在输入中手动补充核心参数,如材质、尺寸、容量、拉链/面料/认证来源。
- AI不确定的视觉判断必须标注【待核验】,不能把猜测写成事实。

### 问题2:标题生成堆砌无逻辑
**现象:** 标题字数超限、关键词重复、读起来像词堆。  
**原因:** 未限制标题长度、核心词位置和重复词规则。  
**处理:**
- 淘宝标题必须控制在30个汉字以内,不得出现重复词。
- 前8字优先放核心词+属性词,后半段补场景词和转化词。
- 生成20个标题测试池时,必须分为搜索流量型和直通车型,不能只换词序凑数量。

### 问题3:运营计划不符合库存与预算
**现象:** AI默认给出大额投流、达人合作、直播等动作,但新品预算或库存承接不了。  
**原因:** 输入端没有限定预算、库存、毛利和履约能力。  
**处理:**
- 输入中尽量补充日预算、库存量、毛利空间、发货能力。
- 若用户给出"日预算100元以内",第15项只能给低预算测款方案。
- 任何投流建议必须写明目标数据和停止条件,不能写"加大投放"这种空话。

### 问题4:主图方案选择困难
**现象:** 5套主图方案都能用,但不知道先做哪套。  
**原因:** 产品定位、价格带和目标用户没有先定死。  
**处理:**
- 高性价比/低价引流优先"容量冲击型"。
- 中端通勤款优先"轻便通勤型"。
- 高客单价/专业款优先"电脑保护型"或对应类目的专业安全型。
- 视觉种草款优先"高颜值种草型"。
- 不确定时只选2套做AB测试,不要让设计同时做5套完整稿。

### 问题5:详情页5屏顺序被随意打乱
**现象:** 用户想把卖点、参数、售后随意提前,详情页变成资料堆砌。  
**原因:** 没有区分"用户决策心理"和"厂家资料顺序"。  
**处理:**
- 默认坚持:心智定位 → 痛点共鸣 → 核心解法 → 场景延伸 → 消除顾虑。
- 高客单价品可先讲核心解法建立信任,但必须说明原因。
- 冲动消费品可把场景延伸前置,但不能省略消除顾虑。
- 任何顺序调整都必须写明:为什么调整、调整后的心理目标、对应证据。

### 问题6:AI把运营表达写成事实承诺
**现象:** 把"更舒服"写成绝对承诺,或把"防泼水"写成"完全防水"。  
**原因:** 确定事实、运营表达、高风险内容没有分层。  
**处理:**
- 确定事实:材质、尺寸、颜色、包装清单,必须忠于资料。
- 运营表达:场景、痛点、搭配建议,可以生成草稿。
- 高风险内容:功效、认证、承重、食品接触、儿童适用,必须人工复核。
- 涉及安全、功效、认证、售后承诺时,统一标注【人工复核】。

## 十三、常见错误与纠正

| 错误调用 | 问题 | 正确做法 |
|---|---|---|
| "帮我分析一下这个产品" | 缺3个必填项 | 补充:产品名称+类目+平台 |
| 同时跑2个产品 | 一条消息只能跑1个 | 跑完第1个再说"同样流程跑第2个" |
| 调用里夹其他任务 | 任务混淆,拒绝执行 | 分开两条消息 |
| 只给品牌名没给链接 | 竞品数据精度低 | 给具体商品链接(淘宝/拼多多/京东) |
| 产品图一次只给1张 | 主图策划精度低 | 至少给:正面/背面/内部/上身/细节 5张 |
| 不给预算却要30天计划 | 计划容易偏大 | 补充日预算/库存/毛利,或接受预算分档方案 |
