AI 电商上架工作流
真正可执行的新品上架流程

从一份产品资料到一份可直接落地的运营包。

Skill 验证输入、自动调用 companion skill 拉数据、把证据与假设分离，输出运营、设计、客服、决策四个角色都能直接使用的内容。

输入产品、类目、平台、定价、产品图必填
检查模式、数据工具、风险、证据等级预检
输出15 模块 + 角色交付包可复核
1

快速诊断

先判断这个品值不值得做，再决定要不要全量跑。

2

完整上架

跑完整 15 项流程，产出角色交付包。

3

素材制作

专注卖点、主图、详情页、标题、生图 Prompt。

4

上架后优化

复盘点击/转化、标题、评论和客服话术。

只有来源可见，数据才有用。

A
用户截图、表格、后台导出、产品事实。
B
授权工具导出或公开页面带链接和时间的观察值。
C
搜索结果信号 + 类目经验。
D
强假设，需人工复核才能转为可用。

生图 Prompt

主图 / 详情场景 / 对比图 / 细节图 Prompt。

确认模型、尺寸、参考图、张数、文字策略、保存路径。

视频分镜

主图视频 / 短视频 / 详情视频 / 仅分镜。

钩子、时间轴、分镜、字幕、运镜、禁止改动。

合规护栏

AI 图和视频只是创意参考，不能直接当上架素材。

不许虚假承诺、不许非授权 logo、产品外观不可失真。

最终交付按角色拆分。

运营、设计、客服、决策各拿一份聚焦包，不只是一份长报告；缺失数据保留为【待核验】。
