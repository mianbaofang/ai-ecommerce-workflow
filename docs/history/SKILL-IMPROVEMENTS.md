# Skill 完善建议清单

## 一、核心问题（必须修复）

### 1. 缺少执行触发机制
**问题:** SKILL.md 写了触发句，但没有明确的执行流程控制
**影响:** AI可能无法自动按14项顺序执行
**解决方案:**
```markdown
## 执行流程（强制顺序）
当识别到触发句后，必须：
1. 先验证3个必填项（产品名称+类目+平台）
2. 明确告知用户："即将按14项标准输出，预计3-8分钟"
3. 逐项执行，每完成1项输出进度（✓ 1/14 产品定位完成）
4. 如遇缺失信息，标注"【待核验】"而不是停止
5. 全部完成后输出4角色分工
```

### 2. 竞品数据获取能力缺失
**问题:** skill提到"web_search / web_fetch 可选"，但没有明确如何调用
**影响:** 竞品分析全靠模型常识，"待核验"标签会很多
**解决方案:** 添加工具调用指引
```markdown
## 工具协同（优先级顺序）
### 竞品数据获取（第5项执行时）
1. **优先:** 如果用户提供竞品链接 → 使用 web_fetch 抓取价格/评论
2. **次选:** 用户只提供品牌名 → 使用 web_search 搜索"{品牌} {产品类目} 淘宝"
3. **兜底:** 无竞品信息 → 基于常识生成 + 标注【待核验-需用生意参谋核实】

### 主图生成（第7项完成后）
- 如进入生图分支 → 先让用户选择公开或自有生图工具,再确认尺寸/参考图/张数等参数
- 生成时必须注明：**此图仅作创意参考，不可直接上架**
```

### 3. 输出格式不统一
**问题:** 14项输出格式混杂（表格/列表/段落），AI容易跑偏
**影响:** 不同AI执行输出格式差异大
**解决方案:** 强制输出格式

## 二、重要优化（建议实施）

### 4. 缺少错误处理机制
**场景:** 用户只说"帮我分析这个包"（缺必填项）
**当前问题:** skill没说AI该怎么办
**建议添加:**
```markdown
## 输入验证与补全
### 缺失必填项时
输出格式：
"""
⚠️ 缺少必填信息，请补充：
[ ] 产品名称 + 一句话描述
[ ] 产品类目
[ ] 目标平台

您可以：
1. 直接复制 templates/user-input-form.md 填写
2. 口述以上3项，我来格式化
"""

### 缺失推荐项时
先标注缺失项，继续执行：
"""
✓ 已接收必填信息，开始执行
⚠️ 以下信息缺失，输出精度会降低：
- 产品图（主图策划/设计Brief精度-30%）
- 成本/售价（价格带判断标注【待核验】）
- 已知竞品（竞品分析基于常识推测）

是否继续？[Y 继续 / N 等我补充]
"""
```

### 5. 平台适配规则不完整
**问题:** 只列了淘宝/拼多多/亚马逊，但没说抖音/快手/1688
**建议添加:**
```markdown
## 平台差异化规则（第9项标题 / 第10项关键词）
| 平台 | 标题特征 | 关键词策略 |
|---|---|---|
| 淘宝 | 30字内，前8字是核心词 | 搜索词覆盖>场景词 |
| 拼多多 | 偏价格+场景，可用"包邮" | 价格词+场景词并重 |
| 抖音 | 场景化+口语化，"XX必备" | 兴趣词>搜索词 |
| 亚马逊 | 完整语法，前80字符权重高 | 英文同义词扩展 |
| 快手 | 方言化+亲切感 | 老铁/家人友好词 |
| 1688 | 批发导向，标注起批量 | 行业术语+规格 |
```

### 6. 差异化机会判断逻辑弱
**问题:** 第6项"差异化机会"没说AI该怎么判断
**当前:** 可能输出空话（"可以做高端市场"）
**建议添加判断框架:**
```markdown
## 第6项执行标准：差异化机会（3-5个）
### 判断框架（必须符合2/3条）
1. **空白带检验:** 竞品在该价格带<5个，或主图/详情页同质化>70%
2. **用户痛点覆盖缺口:** 评论高频痛点在竞品详情页未被处理
3. **场景细分机会:** 通用场景可拆分（通勤→通勤+出差两用）

### 输出格式（每个机会必须包含）
| 差异化方向 | 为什么是机会 | 如何打 | 风险 |
|---|---|---|---|
| 不卖"大容量"，卖"通勤+短途两用" | 竞品都强调容量参数，无场景画面 | 主图1直接放通勤场景，详情页按天数拆解 | 需实拍场景图，成本+500 |
```

## 三、体验优化（可选）

### 7. 增加进度可视化
```markdown
## 执行时进度输出
✓ 1/14 产品定位 [===>              ] 7%
✓ 2/14 用户画像 [======>           ] 14%
⏳ 3/14 核心卖点 [========>         ] 21% 正在翻译用户语言...
```

### 8. 添加质量自检
```markdown
## 输出完成后自检清单
执行完14项+4角色后，AI必须自检：
- [ ] 竞品是否分了至少3个价格带？（第5项）
- [ ] 主图1是否只解决"让用户停下来"一件事？（第7项）
- [ ] 详情页是否按用户疑虑顺序排列？（第8项）
- [ ] 每个结论是否带"为什么"？（全局）
- [ ] 差评预警是否反推到详情页/客服话术？（第11/13项）
- [ ] 角色分工是否可执行（有具体数字/品牌/动作）？（4角色）

如有1项未通过 → 输出警告并重新生成该项
```

### 9. 增加示例触发
```markdown
## 快速示例调用
用户可输入：
- "给我看通勤包的完整示例" → 直接输出 examples/commute-backpack.md
- "我想看主图策划怎么写" → 跳转到 examples/commute-backpack.md 第7项
- "价格带怎么分" → 输出 references/price-band-analysis.md
```

## 四、安全边界补充

### 10. 数据真实性声明（必须前置）
建议在每次执行开始前输出：
```markdown
⚠️ 数据真实性声明
本次分析中，以下数据标注【待核验】：
- [ ] 竞品价格（需用生意参谋/蝉妈妈核实）
- [ ] 用户评论（需实际抓取店铺评论）
- [ ] 关键词搜索量（需用直通车/生意参谋验证）

AI生成的图片仅作创意参考，上架图必须：
1. 实拍产品图
2. 有授权的素材
3. 符合平台规范（无PS痕迹/无违禁词）
```
